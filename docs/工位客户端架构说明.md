# 工位客户端架构说明（app-workstation-client）

> Electron 工位客户端：双屏 webview 内嵌 admin/frontend 的 `/OPWeb`，内置本地配置页，内置测量数据采集服务，通过 BFF 对接 EADAF。

---

## 1. 总体架构

```
app-workstation-client (Electron)
├── 主进程 (main.ts)
│   ├── 窗口管理：主窗 (display0) + 副窗 (display1) 双屏全屏
│   ├── 配置管理：electron-store (OP_BASE / SSO / 工位 / 习惯)
│   ├── 采集服务：serialport / node-hid → 原始帧 → BFF /api/op/ingest
│   ├── IPC：主副窗联动、配置热更新
│   └── 预加载 (preload.ts)：contextBridge 暴露 electronAPI
├── 渲染进程 A：主窗 webview → {OP_BASE}/OPWeb?screen=main
├── 渲染进程 B：副窗 webview → {OP_BASE}/OPWeb?screen=secondary
└── 渲染进程 C：配置页（内置 React 页，独立 bundle）
```

### 数据流
```
[测量工具] --serial/USB--> [采集服务] --原始帧+元数据--> [BFF /api/op/ingest]
                                                              └--> [EADAF ingest 管道]
[OPWeb (webview)] <--REST/WS--> [BFF] <--应用token--> [EADAF 业务数据 API]
[主窗 OPWeb] <==BroadcastChannel 'fmms-op'==> [副窗 OPWeb]  (同源 webview 内通信)
```

## 2. 双屏窗口管理

### 2.1 显示器检测与窗口分配
- 启动时 `screen.getAllDisplays()` 检测显示器
- **双显示器**：主窗→display0 全屏；副窗→display1 全屏（无菜单/无边框）
- **单显示器**：主窗正常显示；副窗可选隐藏或小窗浮动（配置项 `dual_screen`）
- 副窗跟随主窗：主窗最小化/恢复时副窗联动

### 2.2 webview 加载
```
主窗:  loadURL(`${OP_BASE}/OPWeb?screen=main`)
副窗:  loadURL(`${OP_BASE}/OPWeb?screen=secondary`)
```
- OP_BASE 来自 electron-store 本地配置
- 若 OP_BASE 未配置 → 先打开配置页

### 2.3 主副屏同步
主副窗加载同一 `/OPWeb`（同源），通过 **BroadcastChannel `fmms-op`** 通信：
```js
// 主屏选中工卡时广播
channel.postMessage({ event: 'selectCard', workCardId: '...', workstationType: '...' });
// 副屏监听，据此渲染对应工作台
channel.onmessage = (e) => { ... };
```
消息类型：`selectCard`（选工卡）、`selectPart`（选零件）、`syncSlot`（分拣槽位变更）、`navigate`（页面切换）。

## 3. 本地配置与习惯存储（electron-store）

### 3.1 配置结构（LocalConfig）
```ts
{
  op_base_url: "http://localhost:5181",   // admin/frontend OPWeb 地址
  eadaf_frontend_url: "http://localhost:9527",  // SSO 登录入口
  sso_application_id: "10000000-0001-4000-8000-000000000006",
  selected_workstation_code: "OP02-1",     // 上次选择的工位
  selected_workstation_type: "inspection_small",
  dual_screen: true                         // 双屏模式
}
```

### 3.2 用户习惯（UserHabits）
```ts
{
  recent_work_card_ids: ["..."],  // 最近打开的工卡
  page_size: 20,
  sound_alarm: true,              // 声音报警
  last_secondary_tab: "inspection" // 副屏上次标签页
}
```

### 3.3 OPWeb 访问本地配置
preload.ts 通过 `contextBridge` 暴露 `window.electronAPI`：
```ts
window.electronAPI.getLocal()       // 读 LocalConfig + UserHabits
window.electronAPI.setLocal(patch)  // 写入（工位选择/习惯更新）
window.electronAPI.onConfigChanged(cb)  // 配置变更通知
```
OPWeb 中若 `window.electronAPI` 存在（Electron 环境）则用本地存储；否则降级 localStorage（纯浏览器开发）。

### 3.4 配置页（内置 React）
独立 Vite 入口，antd 表单：
- admin/frontend 地址（OP_BASE）
- EADAF 前端地址（SSO）
- SSO 应用 ID
- 测试连接按钮（ping OP_BASE /auth/check）
- 保存后热更新 webview 地址（无需重启）

## 4. 测量数据采集服务

### 4.1 设计原则
- **Electron 只读原始帧 + 转发**，不做协议解析（解析由 EADAF 采集管道或后续本地解析器）
- 不持有 app_secret（安全）；用 webview 中的用户 token 认证
- 可插拔驱动架构：serial / USB HID（P1）、Modbus / TCP（预留）

### 4.2 采集流程
```
1. 启动读 MeasuringTool 配置（workstation_code 关联的工具，protocol/port 等）
2. 按协议加载驱动，打开连接（serialport.open / HID.open）
3. 监听数据 → 组装 CollectFrame { tool_code, workstation_code, raw(base64), ts, ... }
4. 取用户 token（从主窗 webview session / 注入）→ POST BFF /api/op/ingest
5. BFF 回显 measured_value/result → WS collect.frame → 副屏检测表自动填充
6. 监控连接状态 → WS tool.status / tool.alarm
```

### 4.3 驱动接口
```ts
interface CollectorDriver {
  connect(config: MeasuringTool): Promise<void>;
  onFrame(cb: (raw: Buffer) => void): void;   // 原始字节回调
  disconnect(): Promise<void>;
  getStatus(): { online: boolean; lastError?: string };
}
```
- `serial.ts`：`serialport` 库，监听 `data` 事件
- `usbHid.ts`：`node-hid` 库，监听数据
- `modbus.ts` / `tcpCustom.ts`：预留接口，P2 实现

### 4.4 用户 token 获取
采集请求需要用户认证。方案：
- 主窗 webview 登录后 token 在 localStorage；
- Electron main 进程通过 `mainWindow.webContents.executeJavaScript('localStorage.getItem("token")')` 读取
- 或 preload 在 OPWeb 登录成功时 IPC 通知 main 进程缓存 token
- 采集请求用该 token 调 BFF `/api/op/ingest`

## 5. SSO 集成（Electron 透传）

SSO 在 **webview 内完成**，Electron 不参与协议：
1. OPWeb 的 `/auth/login` 按钮 → `window.location.href = {EADAF前端}/auth/login?app={appId}`
2. 用户在 EADAF 登录
3. EADAF 回调到 `{OP_BASE}/auth/callback?token=...`
4. OPWeb 的 AuthCallback 组件捕获 token 存 localStorage → 跳 `/OPWeb`

Electron 仅需：
- webview 允许跳转到 EADAF 域名（`webPreferences` 不限制）
- 允许 localStorage/cookie 持久化（登录态保持）
- 不持有任何 app_secret

---

## 6. 安全

- `contextIsolation: true`，`nodeIntegration: false`
- 敏感操作（串口/HID/文件）仅在 main 进程，通过 `contextBridge` 受控暴露
- 不在任何渲染进程暴露 app_secret
- webview 限制可导航域名（OP_BASE、EADAF 域名白名单）

---

## 7. 打印（标签/委外单）

P1 方案：webview 内 `window.print()` + 标签 HTML 模板。
- 分类条码：生成 Code128/QR 的 SVG/Canvas → 嵌入打印 HTML
- 委外单：表格 HTML
- 调用：`window.electronAPI.print(html)` → main 进程 `BrowserWindow.webContents.print()`
- 特定标签机驱动适配视现场设备（P2 可换系统级打印或串口标签机协议）

## 8. 目录结构

```
app-workstation-client/
├── electron/
│   ├── main.ts               # 主进程入口，窗口/生命周期
│   ├── preload.ts            # contextBridge 暴露 electronAPI
│   ├── windows/
│   │   ├── mainWindow.ts     # 主窗 (display0)
│   │   ├── secondaryWindow.ts# 副窗 (display1)
│   │   └── configWindow.ts   # 配置页窗口
│   ├── config/
│   │   └── store.ts          # electron-store 封装
│   ├── collector/
│   │   ├── index.ts          # 采集服务管理器
│   │   ├── drivers/
│   │   │   ├── serial.ts
│   │   │   ├── usbHid.ts
│   │   │   └── modbus.ts     # 预留
│   │   └── pipeline.ts       # 原始帧→BFF 转发
│   └── ipc.ts                # IPC 处理 + 主副联动
├── src/                      # 配置页前端 (React + antd)
│   ├── ConfigPage.tsx
│   └── main.tsx
├── vite.config.ts            # 双入口 (electron main + config renderer)
├── electron-builder.yml      # 打包配置
├── tsconfig.json
└── package.json
```

## 9. 开发与调试

- `pnpm dev`：启动 Electron（main 用 tsx 监听重启；renderer 用 Vite dev）
- 开发时 OP_BASE 指向 admin/frontend dev server (`http://localhost:5181`)
- 副屏可在单显示器用 `--small-secondary` 参数开小窗调试
- 采集模拟：内置 mock driver（定时发随机字节），无真实硬件时可联调
