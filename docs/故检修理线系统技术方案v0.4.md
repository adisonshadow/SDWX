# 故检修理线配套软件系统技术方案

## 一、需求理解

#### 1.1 项目背景

车间已通过 PCS 系统实现数字化工卡管理，但是在故检修理产线现场，从分拣、故检、修理到收发的全链路仍存在多处数字化断点：工卡中的检测数据仍主要依赖操作员从测量工具读数后手工填写；修理过程、委外与报废处置缺乏与修理设备联动的标准化系统留痕；形迹托盘码盘齐套依赖人工核对；故检不合格后的复检、复修循环缺乏系统管控；零件流转追溯困难、异常呼叫响应滞后。为提升故检修理产线执行环节的数字化水平，亟需建设一套与 PCS 协同、覆盖分拣-故检-修理-收发全工位，集测量工具与修理设备集成、检测数据自动采集、修理过程管控、形迹托盘数字化管理、质量追溯于一体的产线执行管理系统。

#### 1.2 业务痛点分析

- **工卡数据填写低效**：PCS 侧工卡已数字化，但故检环节测量数据需依赖测量工具人工读数抄录，速度慢、易出错，无法与测量工具联动
- **数据链路未贯通**：测量工具与工卡脱节，检测数据不能自动采集、判异；修理设备与工卡脱节，修理、委外、报废处置过程无系统化记录，执行结果难以及时汇总
- **修理流程不闭环**：故检不合格后的修理、复检、复修循环缺乏工位级一键流转的能力，委外件、报废件分流不规范，易出现漏检、错流
- **形迹托盘管理粗放**：形迹托盘内衬卡槽无数字化映射，齐套性校验效率低、易出错
- **追溯能力弱**：零件出入库、检测、修理、委外记录分散，质量问题追溯困难
- **异常响应不及时**：现场测量工具或修理设备故障、工艺疑问等呼叫无统一推送渠道，产品技术员与操作员(组长)无法实时感知现场异常



#### 1.3 建设目标

建设一个**工位侧执行管理系统**——**紧贴分拣-故检-修理-收发一线作业与现场操作人员**，不替代 PCS 等上层计划与管控职能；依据本产线业务特点，**有选择地整合 PCS、PCS-AGV 物流等上层与外围系统能力**，聚焦工卡落地执行与现场数据采集；**以工位操作便捷为核心设计原则**，采用桌面客户端、主副屏协同与一键式操作，降低多系统切换与现场作业负担。在此基础上，具体建设目标如下：

1. 承接 PCS 数字化工卡，实现分拣、故检、修理、收发全工位的接收、执行、回填与完工报工全流程可视化
2. 集成各类测量工具，实现检测数据自动采集、自动填入工卡、公差判异
3. 打通与 PCS 生产控制系统的数据链路：PCS 下发含完整工位、物料、WBS 工艺信息的工卡至本系统服务端，由服务端分发至工位客户端
4. 通过集成 PCS-AGV 物流模块实现零件转运一键申请、状态实时跟踪，本系统仅作为调用入口
5. 建立形迹托盘存料数字化模版管理体系，实现分拣工位可视化码盘与自动齐套校验
6. 覆盖修理全流程管控，支持修理执行记录、委外单拆分打印、报废登记，形成故检-修理-复检的业务闭环
7. 建立故检线边库数字化台账，覆盖待委外缓存、零件暂存（待处理）、报废三类库存，实现零件出入库闭环管理与全流程追溯
8. 提供产线实时监控大屏，**用于生产状态可视化监控**
9. 建设异常呼叫通知体系，通过专属接收客户端实现现场异常向技术员、操作员(组长)的实时推送通知



#### 1.4 系统定位

本系统定位为故检修理产线的**执行层管理系统**，上接 PCS 生产控制系统（不直连车间仓储控制管理系统WES、不直连车间AGV调度系统、不直连其他外部系统），下连各类测量工具及修理设备，承上启下实现生产指令的落地执行、生产数据的实时采集与工卡执行结果回传，是连接计划层与工具设备层、覆盖分拣-故检-修理-收发全工位作业的关键枢纽。

## 二、关键业务流程



#### 2.1 整体业务流程

故检修理产线的核心业务流程围绕「工卡驱动、分工作业、闭环流转」展开，覆盖从工卡下发到完工入库的全链路过程，包含故检-修理-复检的循环闭环，以及合格入库、委外、报废三类分流路径。

**主流程主线**：PCS 下发完整工卡至本系统服务端 → 服务端按工位属性分发至对应工位客户端 → 分拣工位清点码盘、齐套校验 → 故检工位检测判异 → 合格件经修理工位流转至收发工位入立库 / 不合格件流转至修理工位 → 修理工位执行修理/委外/报废处置 → 修理完成 PCS 增复检工卡（待定） → 故检工位复检 → 最终合格件经收发工位入立库缓存。

#### 2.2 工卡接收与分发流程

本流程承接 PCS 已下发的数字化工卡，由**服务端统一接收**后按工位属性分发至对应工位客户端，确保各工位只处理与本工位类型匹配的工卡。各工位作业完成后，由服务端统一向 PCS 上报该工位的**工卡执行结果**。

**（1）PCS 下发工卡内容要求**

PCS 向本系统服务端推送的工卡须包含完整信息：


| 数据类别   | 包含内容                                                            |
| ------ | --------------------------------------------------------------- |
| 工卡基础信息 | 任务号、架次、计划时间等                                                    |
| 工位归属   | 工卡所属工位类型：分拣 / 故检1 / 故检2 / 修理 / 收发                               |
| 物料清单   | 零部件编码、数量；**是否默认委外**及委外方；**是否只换不修**                              |
| WBS 信息 | 本产线相关的工艺要求、检测项、公差标准；工艺文档/文件/视频的**访问地址**（不含其他产线信息）；视频地址须支持客户端内播放 |


本系统向 PCS 提供 **WBS 标准结构**规范，PCS 下发工卡时按该结构组织 WBS 树形数据。字段定义、`inspectionItems` 与 `processResources` 结构及 JSON 示例见 **3.4.5 节「PCS 下发工卡中的 WBS 信息内容」**。

**（2）分发与执行步骤**

1. PCS 通过 TCP/IP 接口向本系统服务端推送数字化工卡
2. 服务端接收后按工位属性自动分发至分拣、故检1、故检2、修理、收发对应工位的客户端
3. 客户端实时接收工卡，按「未接受 / 待处理 / 处理中 / 已完成」四种状态分类展示
4. 操作员查看任务详情，确认后一键接受任务，系统记录接受时间与操作人员（本系统内部）
5. 工位作业全部完成后，操作员提交工卡执行结果，由服务端汇总后**上报 PCS**

```mermaid
sequenceDiagram
  participant PCS
  participant Server as 本系统服务端
  participant Client as 工位客户端
  PCS->>Server: 推送完整工卡(工位/物料/WBS)
  Server->>Client: 按工位类型 WebSocket 分发
  Client->>Client: 接受任务/执行作业(内部状态)
  Client->>Server: 提交工卡执行结果
  Server->>PCS: 上报该工位工卡执行结果
```





#### 2.3 分拣作业流程（OP01）

分拣工位负责小零件清点、码盘与齐套校验，通过形迹托盘数字化模版实现可视化码盘与齐套判定。

```mermaid

    direction LR
    A[清洗后小件提篮<br/>经传输机至分拣位] --> B[AGV 配送空形迹托盘<br/>至分拣工位] --> C[操作员接收分拣工卡]
  end
  subgraph s2 [② 可视化码盘与齐套]
    direction LR
    D[副屏展示<br/>形迹托盘存料模版] --> E[码放零件并<br/>toggle 卡槽状态] --> F{齐套校验}
    F -->|未齐套| E
  end
  subgraph s3 [③ 完工与转运]
    direction LR
    G[提交分拣执行结果<br/>上报 PCS] --> H[发起 AGV 转运<br/>至故检工位]
  end
  C --> D
  F -->|已齐套| G
```



1. 清洗后小零件提篮通过传输机送至分拣位；空形迹托盘经 AGV 配送到分拣工位
2. 操作员通过产线管理系统客户端接收 PCS 下发的分拣工卡
3. 主屏选中工卡后，副屏自动加载对应形迹托盘存料模版
4. 操作员将零件放入卡槽，点击副屏半透明形状切换「有/无」状态，有料时形状高亮
5. 系统实时比对工卡物料清单与卡槽状态，全部就位后弹出齐套提示
6. 确认齐套后提交分拣执行结果，由服务端上报 PCS
7. 通过客户端发起 AGV 物流申请（经 PCS-AGV 模块），将形迹托盘转运至故检工位



#### 2.4 故检作业执行流程（OP02）

**（3）故检作业步骤**

1. 操作员进入工卡详情页，查看零部件清单、工艺要求及检测公差标准（来自 WBS）
2. 扫码枪扫描零部件条码，系统定位并高亮对应检测任务
3. 副屏检测执行界面：
  - **数字量具检测项**：自动采集测量数据，依据工艺公差自动判异
  - **人工判定项**：工艺标注的难测项由人工判定合格/不合格
  - **拍照留痕**：工艺注明需拍照的不合格项，界面提示拍照；踩踏踏板或点击按钮拍照，支持预览与重拍
4. 不合格零件处置分类（三类）：
  - **委外修理**：由物料「默认委外」属性驱动，系统自动建议
  - **修理**（厂内）：需在本厂修理工位处置的零件
  - **报废**：含「只换不修」等策略对应的报废件。在故检工位确认的“报废件”，需要向车间PCS系统发起缺件申领
    报废件产生下列情形：
    1）.	“必换件”，工艺规定。
    2）.	“报废件-1”，工艺未规定，但零件寿命问题。
    3）.	“报废件-2”，故检检查后确认。
    4）.	“报废件-3”，修理后复检确认。

5. 确认后生成分类流转条码并打印，分类流转至修理工位或收发工位
6. 全部零件检测完成后提交故检执行结果，由服务端上报 PCS



#### 2.5 修理与处置作业流程（OP03）

本流程覆盖厂内修理、委外处置、报废登记三类场景，支持故检-修理-复检的循环闭环。

```mermaid
flowchart TB
  subgraph s1 [① 接收与三路处置]
    direction LR
    A[接收修理工卡] --> B[副屏处置操作]
    B --> C1[修理执行] & C2[委外操作] & C3[报废登记]
  end
  subgraph s2 [② 委外支线]
    direction LR
    C2 --> I[按委外方拆分委外单打印] --> J[人工持单至委外系统录入]
  end
  subgraph s3 [③ 完工上报与复检触发]
    direction LR
    C1 --> D{全部零件有处置结果?}
    C3 --> D
    C2 --> D
    D -->|否| B
    D -->|是| E[点击处置完成] --> F[上报 PCS 修理执行结果] --> G[PCS 生成复检工卡] --> H[服务端分发至故检工位]
  end
```



1. 操作员接收修理工卡，查看待处置零件清单与不合格原因
2. 副屏支持三类操作：
  - **修理执行**：选择维修方法、修理设备及工装，填写修理结论
  - **委外操作**：系统按物料属性默认勾选委外零件，支持手工增删；提交后生成唯一委托单编码，按委外方拆分为多张委外单，打印预览（委外系统物理隔离，人工持纸质单录入）
  - **报废登记**：多选零件报废，可选填备注
3. 工卡内所有零件均有执行结果（修理完成/委外/报废）后，「处置完成」按钮可用
4. 点击提交，修理执行结果由服务端上报 PCS
5. PCS 接收后生成复检工卡，重新下发至故检工位（待定），进入复检循环，直至合格、报废或委外出厂



#### 2.6 收发与线边库作业流程（OP04）

收发工位负责产线与立库、委外环节的交接，通过线边库实现零件缓存与分类管理。

**（1）小零件收发（OP04-1）**

```mermaid
flowchart TB
  subgraph s1 [① 入库分流]
```

**业务说明**：

- **合格小零件入立库缓存（未齐套）**：故检合格但尚未与其他零件齐套的小零件，经收发工位登记后 AGV/人工入立库缓存。
- **出库至产线各工位**：原则上合格的小零件均须**先入立库**，再由立库发往产线各工位；除紧急「串件」外（实物可能直接转装配线，但账务须回传 PCS 与 WES），具体作业方式须与车间确认后落地。

**（3）线边库三类库存**


| 库存类型      | 说明            |
| --------- | ------------- |
| 待委外缓存     | 等待委外发运的零件     |
| 零件暂存（待处理） | 产线各环节待继续处理的零件 |
| 报废        | 报废件，须执行报废出库动作 |


收发工位作业完成后，由服务端将工卡执行结果上报 PCS；线边库出入库流水在本系统内部记录。

#### 2.7 测量工具接入与数据采集流程

1. 新测量工具接入故检工位终端后，操作员在客户端「测量工具注册」界面扫描发现并注册
2. 选择工具类型，输入工具编号，完成注册；注册信息同步至服务端
3. 系统按工具类型加载驱动，通过串口 / Modbus RTU / Modbus TCP 等与测量工具通信
4. 故检作业时自动匹配测量工具，实时采集测量数据
5. 数据解析校验后写入工卡检测项并本地缓存，网络恢复后补传服务端
6. 持续监控测量工具在线状态，异常时弹窗报警并记录故障日志，支持呼叫技术员处理



#### 2.8 AGV 物流调用流程

本系统**不直接对接 AGV 调度系统**，所有 AGV 能力通过 **PCS-AGV 物流模块**实现，本系统仅作为工位侧调用入口。

#### 3.2 工位客户端功能

工位客户端部署于故检修理产线各工位终端，是操作员日常作业的主要入口。系统承接 PCS 经服务端下发的数字化工卡，将工卡执行、测量工具采集、修理设备操作、物流转运、线边库记账、异常呼叫等现场操作集中在一个桌面应用中，减少多系统切换与人工抄录；各工位根据配置仅接收与本工位类型匹配的任务，主屏负责工卡与通用操作，副屏承载分拣码盘、故检检测、修理处置或线边库等专属作业界面。

**（1）通用基础功能**

**a）工位配置**

客户端首次部署或产线工位调整时，可通过配置界面指定当前终端所属工位类型，可选值为：**分拣、故检1（小件）、故检2（大/小件）、修理、收发**。配置生效后，客户端仅接收并展示对应工位的工卡任务与功能菜单，避免操作员错接其他工位任务，也便于服务端按工位属性精准分发 PCS 下发的工卡。

**b）员工登录**

支持工号密码登录，绑定操作人员身份。工卡接受、检测录入、修理记录、出入库记账、异常呼叫等关键操作均自动关联当前登录人，作为质量追溯与审计留痕的依据。

**c）工卡全生命周期可视化管理**

工卡是产线作业的组织单元。客户端以 PCS 数字化工卡为数据源，在工位侧提供从接收、接受到执行、完工的全流程可视化管理。界面采用卡片式列表，按任务状态分色展示，操作员无需翻阅 PCS 即可直观掌握本工位待办任务，快速进入作业。

- **状态标识**：每个零部件独立展示检测或处置状态（待处理/检测中/已完成/不合格），并以「合格/不合格」等样式直观呈现最终结论；多零件并行作业时，不同状态一目了然，减少误判与重复作业

- **检测结果统计**：按零部件汇总检测或处置完成情况，如「检测完成（3合格、1不合格）」；操作员可快速判断本工卡是否具备提交执行结果的条件

**f）测量工具状态监控与异常报警**

测量工具是自动采集链路的关键环节，工具断连或数据异常将直接影响工卡回填与执行结果完整性。客户端对已注册测量工具实施运行状态持续监测，在异常发生的第一时间提醒操作员，并留存故障记录，便于维护人员快速响应。

![测量工具状态监控与异常报警界面](images/image009.png)

- **在线状态监控**：持续检测各已注册测量工具的在线/离线及通信健康状态，在工卡界面或工具监控视图中实时展示；操作员开工前可确认测量工具就绪，避免作业中途才发现连接失败
- **异常报警**：当测量工具断线、通信超时或采集数据异常时，立即弹出报警提示窗，明确故障工具与异常类型；减少静默失败导致的数据缺失，降低不合格品漏检风险
- **故障日志**：自动记录每次测量工具异常的时间、工具编号、故障描述等信息，形成可追溯的故障台账；为维护计划制定、备件管理与产线停机分析提供数据支撑

**g）异常呼叫**

故检修理现场测量工具或修理设备故障、工艺疑问、物料异常等情况时有发生，需要快速触达技术员或操作员(组长)。客户端内置呼叫功能，操作员在不离开工位、不中断当前工卡上下文的前提下即可发起求助，并附带问题类型与简要描述，缩短异常响应时间，降低因等待支持造成的产线停顿。

![呼叫技术员/操作员(组长)界面](images/image010.png)

- **呼叫人员选择**：支持选择技术员、操作员(组长)作为被叫对象；按问题性质定向呼叫，提高响应针对性，避免广播式通知带来的信息淹没
- **呼叫类型**：提供测量工具/修理设备故障、技术求助、人员异常、物料异常四种标准类型；分类信息便于后台统计高频异常与优化预防性维护
- **问题描述**：支持填写简要文字说明，描述现场现象或需求；被叫人员在到场前即可预判所需工具、备件或工艺资料，提升一次解决率

**h）AGV 呼进呼出**

零件在分拣、故检、修理、收发等工位间的流转，直接影响产线节拍。客户端提供AGV 转运申请，与工卡作业流程同步并行，操作员完成本工位作业后无需离开界面、无需电话协调，即可一键发起物流任务，并全程掌握运输状态，实现「作业完成—物流触发—下工位衔接」的顺畅闭环。物流能力由 **PCS-AGV 模块**提供，本系统仅作为调用入口。

![AGV物流进站/出站操作界面](images/image011.png)

- **一键转运申请**：工卡详情页内置「AGV 物流」入口，操作员完成作业后一键触发进站或出站申请；界面操作简单，适合戴手套、高频作业的现场环境
- **任务自动生成**：系统根据当前工位、目标工位、绑定载具及工卡零件清单，自动组装完整转运任务，包含起点、终点、载具编号、零件清单及优先级；减少人工填写差错，缩短申请等待时间
- **状态实时跟踪**：通过 PCS 接口持续接收待执行、运输中、已到达等状态反馈，并在工卡界面同步展示；操作员可掌握物流进度，合理安排下道工序准备
- **到达通知**：AGV 抵达指定工位时，客户端主动弹窗提醒操作员，提示及时进行零件交接；避免靠人工盯守或口头传递造成的等待与积压
- **多载具支持**：支持形迹托盘、形迹小车、周转箱等多种产线常用载具的扫码绑定与转运申请；不同零件状态可对应不同载具类型，适配故检修理线多样化物流场景

- **可视化码盘**：副屏中每个卡槽对应一个半透明形状（圆形/正方形/自定义形状），标注对应物料种类编码；操作员将零件放入卡槽后，点击形状可切换「有零件/无零件」状态，有零件时形状高亮显示，实现账面相符的可视化确认
- **齐套校验**：系统实时比对工卡物料清单与当前卡槽选中状态，全部物料就位后弹出齐套完成提示；未齐套时明确提示缺项，减少带缺件托盘流入故检工位
- **齐套结果上报**：操作员确认齐套后提交分拣执行结果，由服务端汇总后上报 PCS，作为分拣工位完工依据；齐套后可发起 AGV 转运至故检工位

**（3）故检工位专属功能（故检1、故检2，主屏 + 副屏）**

故检1（小件）、故检2（大/小件）均采用主屏+副屏模式。主屏负责工卡与通用功能，副屏为检测执行界面，支持数字量具自动采集、人工判定难测项、拍照留痕与不合格分类打码，覆盖初检与 PCS 下发的复检工卡场景。

- **双模式检测**：可对接数字量具的检测项由系统自动匹配测量工具、采集测量值并依据工艺公差判异；工艺标注的难以测量项由操作员人工选择合格/不合格，兼顾自动化与现场灵活性
- **拍照留痕**：工艺注明需拍照的不合格项，界面显示拍照标识；支持踩踏拍照踏板或点击界面按钮触发拍照，拍照后可预览效果并支持重拍；图片与检测项、零件绑定归档，支撑质量追溯
- **不合格分类打码**：检测完成后，系统根据物料属性（默认委外、只换不修等）自动给出处置建议（委外修理/修理/报废）；操作员确认后生成分类流转条码并支持标签打印，用于零件向修理工位或收发工位的物理分流标识
- **结果统一上报**：全部零件检测完成后统一提交故检工卡执行结果，由服务端上报 PCS；复检场景下若仍不合格，可再次流转修理，直至合格或报废

- **委外操作管理**：系统根据工卡物料属性默认勾选需委外的零件，支持人工新增勾选或取消默认委外；提交后生成唯一委托单编码，按委外方自动拆分为多张独立委外单，提供打印预览与批量打印；委外系统与本系统物理隔离，由操作员持纸质单据至委外系统人工录入
- **报废登记**：支持多选零件批量报废，可选填报废原因与备注，与线边库报废类库存联动记账
- **处置完成校验**：实时统计工卡内零件处置进度，当所有零件均生成执行结果（修理完成/委外/报废）后，「处置完成」按钮高亮可用；点击后统一上报修理工卡执行结果至 PCS，触发 PCS 生成复检工卡

收发工位主屏负责工卡与通用操作，副屏为线边库操作界面，承担产线零件缓存、委外收发、报废出库及与立库交接等职能，是产线物流与库存的枢纽工位。

- **线边库分类管理**：线边库分为三类库存——**待委外缓存**、**零件暂存（待处理）**、**报废**，分区记账、独立统计；不同业务状态零件分区管理，避免混放、错发
- **出入库操作**：扫描载具或零件条码，选择库存类型与货位，确认数量后完成入库/出库记账；报废件须执行报废出库动作并留存凭证
- **委外收发登记**：委外零件出库发运、委外完成返回均通过收发工位登记，留存交接记录；委外完成件可暂存线边库等待故检复检或最终入立库
- **立库交接**：产线处理完成的合格零件、委外最终完成件，经收发工位核验后，发起 AGV 或人工转运存入立库缓存
- **执行结果上报**：收发工位工卡作业完成后，由服务端将工卡执行结果上报 PCS；线边库出入库流水在本系统内部记录

**（4）数据接入与输出**

- **接入日志**：完整记录所有数据接入日志，支持数据溯源与问题排查

**（5）BOP 与 WBS 数据包管理**

**a）WBS 数据管理**

WBS（Work Breakdown Structure）是本系统组织工卡、检测任务与执行数据的核心模型。工卡自 PCS 下发时携带 WBS 树形结构，本系统提供 **WBS 标准结构**规范，PCS 按该规范组织本产线相关 WBS 数据（不含其他产线信息）。文档、文件、视频仅提供访问地址。

本系统负责**故检阶段与修理阶段** L5 工作包的执行数据填充（测量值、判定结论、检测图片、修理记录、处置结论、实际工时等），L1～L4 由 PCS/MES 等上游维护，本系统只读引用。

**（1）五层 WBS 分层逻辑**

编码规则：`WBS-{生产令号}-L{层级}-{序号}`


| 层级  | 名称      | 职责说明                   | 典型示例                                   |
| --- | ------- | ---------------------- | -------------------------------------- |
| L1  | 顶层项目    | 整机大修交付总目标              | `WBS-XXX-123-L1-001`：H130 型飞机左主起落架大修项目 |
| L2  | 全流程大修阶段 | 覆盖拆解→清洗→故检→修理→复装→测试→交付 | `WBS-XXX-123-L2-140`：故检与缺陷评估阶段         |
| L3  | 工序模块    | 独立工序包                  | 入厂清点、尺寸检测、缺陷评估                         |
| L4  | 零部件包    | 按 BOM 拆分的检测包           | 减震支柱组件尺寸检测                             |
| L5  | 最小工作包   | 可派工、可报工的最小单元           | 外筒内径三坐标测量                              |


**（2）L5 工作包核心业务字段**


| 字段                              | 说明         | 示例值                      |
| ------------------------------- | ---------- | ------------------------ |
| wbsCode                         | WBS 唯一编码   | `WBS-XXX-123-L5-142.1.1` |
| name / description              | 工作包名称与内容   | 外筒内径三坐标测量                |
| standardBasis                   | 适航/工艺依据    | H30 CMM 3.2.1 支柱外筒公差表    |
| inspectionTool                  | 指定测量工具     | 桥式三坐标测量机                 |
| planWorkHours / actualWorkHours | 计划/实际工时（h） | 6 / 5.8                  |
| status                          | 执行状态       | 待执行 / 检测中 / 已完成          |
| disposalResult                  | 处置结论       | 合格 / 自修 / 报废 / 委外        |


**（3）PCS 下发工卡中的 WBS 信息内容（对应 2.2 节）**

2.2 节要求 PCS 推送的工卡须携带完整 **WBS 信息**。本系统提供 WBS 标准结构，PCS 按该结构组织数据；**仅包含故检修理产线相关节点**（通常对应 L2 故检与缺陷评估、L2 修理等阶段及其下属 L3～L5），**不得夹带其他产线、其他大修阶段的 WBS 子树**，避免工位客户端展示冗余、误操作无关工序。

工卡内 WBS 信息在业务上须覆盖下表四类内容，本系统自 PCS 接收后只读展示，并在 L5 节点回填执行结果：


| 内容类别       | 承载层级                                                              | 用途说明                                    | 本系统侧行为                                  |
| ---------- | ----------------------------------------------------------------- | --------------------------------------- | --------------------------------------- |
| **工艺要求**   | L3～L5 的 `processRequirement` 或 `description`                      | 描述本工序/本检测包的操作步骤、注意事项、合格判定原则等文字性工艺要求     | 工卡详情页、副屏检测界面展示，供操作员现场查阅                 |
| **检测项**    | L5 的 `inspectionItems[]`                                          | 可执行的最小检测粒度：检测特征、标准值、公差、数采/人工属性、是否需拍照等   | 驱动副屏检测表格、测量工具自动匹配与判异、人工判定项              |
| **公差标准**   | 含于各检测项的 `nominalValue`、`upperTolerance`、`lowerTolerance`、`unit` 等 | 单项测量的合格区间，用于自动判异                        | 采集值与公差比对，不合格高亮                          |
| **工艺资料地址** | 各节点的 `processResources[]`                                         | 工艺文档、附件文件、教学视频的 **HTTP(S) 访问地址**（非文件本体） | 工卡详情提供链接；**视频在客户端内置播放器中直接播放**，无需跳转外部浏览器 |


**产线范围约束示例**：工卡 `workCardId=WC-2026-0618-001` 仅下发 `productionLineCode: "GJJ-XL-01"`（故检修理产线）且 `wbsScope: ["L2-140", "L2-150"]`（故检阶段、修理阶段）下的 WBS 子树；L2 拆解、L2 复装等与本产线无关的阶段节点由 PCS 裁剪，不进入本系统。

**（4）工艺要求**

工艺要求用于承载**文字性、段落性**的作业指导，与结构化检测项互补。推荐字段：


| 字段                 | 类型     | 说明                                              | 示例                                    |
| ------------------ | ------ | ----------------------------------------------- | ------------------------------------- |
| processRequirement | string | L3/L4/L5 工序级工艺要求正文，支持换行                         | 「测量前须校零三坐标测头；每件至少测 3 个截面，每截面 4 点均匀分布」 |
| description        | string | 工作包简述（可与 processRequirement 并存，description 偏概要） | 「测量外筒内孔圆度、圆柱度、导向段磨损量，对比原厂公差限值」        |
| standardBasis      | string | 适航/工艺依据编号                                       | `H30 CMM 3.2.1 支柱外筒公差表`               |


L3/L4 节点可仅含 `processRequirement` 与 `processResources`；**可派工、可填数的检测要求**应下沉至 L5 的 `inspectionItems`。

**（5）检测项与公差标准（inspectionItems）**

L5 工作包下的 `inspectionItems` 数组是本系统故检副屏、自动采集与判异的**直接数据源**，每一项对应工卡检测表格中的一行。推荐结构如下：


| 字段                 | 类型      | 必填       | 说明                                | 示例                   |
| ------------------ | ------- | -------- | --------------------------------- | -------------------- |
| itemCode           | string  | 是        | 检测项唯一编码，工卡内不重复                    | `INS-142.1.1-01`     |
| itemName           | string  | 是        | 检测项名称                             | `导向段内径 A 截面`         |
| feature            | string  | 否        | 检测特征/几何量类型                        | `内径`                 |
| nominalValue       | number  | 数采项必填    | 标准值（工艺值）                          | `120.000`            |
| upperTolerance     | number  | 数采项必填    | 上偏差（相对标准值，正数表示上限偏移）               | `0.050`              |
| lowerTolerance     | number  | 数采项必填    | 下偏差（相对标准值，负数表示下限偏移）               | `-0.020`             |
| unit               | string  | 是        | 计量单位                              | `mm`                 |
| dataCollectionType | enum    | 是        | `auto`：测量工具数采；`manual`：人工判定合格/不合格 | `auto`               |
| inspectionToolCode | string  | auto 时必填 | 指定测量工具类型或编号，用于客户端自动匹配             | `CMM-001`            |
| requirePhoto       | boolean | 否        | 不合格时是否必须拍照留痕                      | `true`               |
| manualOnly         | boolean | 否        | 难测项，仅人工判定、不绑定数采                   | `false`              |
| partRef            | string  | 否        | 关联零部件实例编码，与工卡物料清单对应               | `PART-OUTER-CYL-001` |


**公差判异规则（本系统实现）**：实测值 `actualValue` 满足  
`nominalValue + lowerTolerance ≤ actualValue ≤ nominalValue + upperTolerance` 为合格，否则不合格。  
例：标准值 120.000 mm，上偏差 +0.050、下偏差 -0.020 → 合格区间 **119.980～120.050 mm**。

| itemCode       | itemName   | nominalValue | upperTolerance | lowerTolerance | unit | dataCollectionType |
| -------------- | ---------- | ------------ | -------------- | -------------- | ---- | ------------------ |
| INS-142.1.1-01 | 导向段内径 A 截面 | 120.000      | 0.050          | -0.020         | mm   | auto               |
| INS-142.1.1-02 | 导向段内径 B 截面 | 120.000      | 0.050          | -0.020         | mm   | auto               |
| INS-142.1.1-03 | 外观划伤目视检查   | —            | —              | —              | —    | manual             |


**（6）工艺资料访问地址（processResources）**

工艺文档、附件、视频**不随工卡打包传输**，由 PCS/CAPP 等上游系统提供 **URL 访问地址**，本系统工位客户端按链接拉取或在线打开。推荐挂载在 L3～L5 任意节点（与该项作业最相关的层级）。


| 字段               | 类型      | 必填        | 说明                                                              | 示例                                          |
| ---------------- | ------- | --------- | --------------------------------------------------------------- | ------------------------------------------- |
| resourceId       | string  | 是         | 资料唯一标识                                                          | `CAPP-H30-3210-0201`                        |
| resourceType     | enum    | 是         | `document`：工艺规程/作业指导书（PDF、Word 等）；`file`：图纸、附件；`video`：操作教学视频   | `video`                                     |
| title            | string  | 是         | 展示标题                                                            | `外筒内径三坐标测量操作教学`                             |
| url              | string  | 是         | **产线网可访问**的 HTTP/HTTPS 地址                                       | `https://capp.intra/files/H30-CMM-1421.mp4` |
| mimeType         | string  | 否         | 文件 MIME，便于客户端选择打开方式                                             | `video/mp4`、`application/pdf`               |
| playableInClient | boolean | video 时建议 | 为 `true` 表示须在工位客户端内嵌播放；本系统仅对 `playableInClient=true` 的视频启用内置播放器 | `true`                                      |


**视频地址要求（与 2.2 节、3.2.1 节工卡详情能力一致）**：

- 须为产线执行网（工位终端）**可直接访问**的地址，无需登录外网或跨密级跳转；
- 推荐 **MP4（H.264）** 等 Electron 内置播放器可解码格式；若使用 HLS，须确认客户端播放器能力；
- **不得**依赖浏览器插件、Flash、仅内网 DNS 解析不到的域名；
- PCS 下发时 `playableInClient: true` 的视频，操作员在工卡详情点击后在**客户端内全屏/窗口播放**，不强制打开系统默认浏览器，避免现场切换窗口。

**（10）本系统与 WBS 的衔接关系**

- **自 PCS 接收**：工卡携带 WBS 树；L5 含 `processRequirement`、`inspectionItems`（检测项与公差）、`processResources`（工艺文档/文件/视频 URL）；仅本产线相关子树
- **本系统填充**：故检与修理阶段 L5 执行数据（测量值、判定结论、检测图片、修理记录等）写入节点并追加 dataHistory
- **回传 PCS**：工位工卡执行结果上报时携带 L5 执行摘要，由 PCS 更新数字化工卡
- **与 BOP 联动**：工卡完工时汇总生成 BOP 执行数据包

**b）BOP 执行数据包**

BOP（Bill of Process，工艺过程清单）执行数据包是工卡执行过程中产生的完整数据集合。本系统作为工位侧执行系统，**负责保存和展示故检修理产线相关的 BOP 数据**——即分拣、故检、修理、收发各工位在执行过程中产生或汇聚的工艺与执行记录，供技术员、检验员等角色浏览与质量追溯。


| 数据类别     | 包含内容                            |
| -------- | ------------------------------- |
| 工艺基础数据   | 工卡信息、零部件清单、工艺路线、检测项清单、公差标准、工艺参数 |
| 执行过程数据   | 操作人员、开始/结束时间、各工序执行记录、操作日志       |
| 检测结果数据   | 各项测量值、合格判定结果、测量工具信息、检测图片/视频     |
| 处置结果数据   | 修理记录、委外记录、报废记录、分类流转信息           |
| 测量工具运行数据 | 使用测量工具编号、工具状态、工具运行参数、校准信息       |
| 物料流转数据   | 零部件出入库记录、AGV 转运记录、载具信息          |
| 异常处理数据   | 不合格项记录、返工记录、呼叫记录、异常处理结果         |


- **数据包生成**：工卡执行完成后自动生成 BOP 执行数据包，汇总该工卡在本产线各工位的执行痕迹
- **数据包存储**：采用结构化存储与文件存储相结合的方式，确保数据完整性与查询效率
- **数据包查询**：支持按工卡号、零件号、批次号、时间范围等多维度查询 BOP 执行数据包
- **质量追溯**：基于 BOP 执行数据包实现从入厂分拣、故检、修理到处置完成的本产线全流程质量追溯

**（6）系统配置管理**

- **用户与角色管理**：维护产品技术员、生产准备技术员、故检员、操作员、试验员、检验员、器材保管工、生产准备工等账号与角色权限，支撑客户端登录与通知推送的权限隔离

| PCS → 本系统服务端 | 完整工卡（工位、物料属性、WBS 及资料地址） | TCP/IP 推送 | 实时    |
| 本系统 → PCS    | 各工位工卡执行结果               | TCP/IP 上报 | 实时    |
| 双向           | 心跳、重连                   | TCP/IP    | 每 5 秒 |


```mermaid
sequenceDiagram
  participant PCS
  participant Server as 本系统服务端
  participant Client as 工位客户端
  Note over PCS,Client: 接口1 工卡下发
  PCS->>Server: 推送工卡/复检工卡
  Server->>Client: 按工位分发
  Note over PCS,Client: 接口2 工卡执行结果上报
  Client->>Server: 完成作业提交
  Server->>PCS: 工卡执行结果
  Note over PCS,Client: 接口3 AGV物流调用
  Client->>Server: 转运申请
  Server->>PCS: 提交 PCS-AGV 任务
  PCS-->>Server: 任务状态
  Server-->>Client: 展示物流进度
```



**a）核心接口说明**

1. **工卡下发接口**：PCS → 服务端，含工位、物料（默认委外/委外方/只换不修）、WBS（本产线工艺与资料 URL）；服务端 → 客户端分发
2. **工卡执行结果上报接口**：本系统 → PCS，各工位完工时上报检测/处置明细、操作人员、时间等
3. **AGV 物流调用接口**：本系统 → PCS 提交转运申请，PCS 回传任务状态

**（2）与测量工具对接方案**

工位客户端直连测量工具，数据写入本系统，随工卡执行结果一并体现在上报 PCS 的工卡结果中。

**a）支持的测量工具类型**

几何量（气动量仪、三坐标、轮廓仪等）、光学测量、力学性能及定制测量工具。

**b）支持的通信协议**


| 协议类型           | 物理接口   | 适用场景     |
| -------------- | ------ | -------- |
| 串口 RS232/RS485 | DB9、端子 | 传统测量工具   |
| Modbus RTU     | RS485  | 工业现场测量工具 |
| Modbus TCP     | 以太网    | 智能测量工具   |
| TCP/IP 自定义     | 以太网    | 高端测量工具   |
| USB HID        | USB    | 便携测量工具   |


**（3）AGV 物流调用说明**

本系统**不直接对接 AGV 物流系统**，所有调度通过 **PCS-AGV 物流模块**实现。

- **调用链**：工位客户端 → 服务端 → PCS → 车间仓储管理 → AGV 调度
- **功能边界**：本系统负责申请提交、状态展示、到达提醒；不涉及路径规划与车辆管理
- **优势**：复用车间 PCS-AGV 统一调度体系
