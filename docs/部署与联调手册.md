# 部署与联调手册

> 本地开发启动顺序、EADAF/Electron 联调、Mock 种子数据说明。

---

## 1. 前置条件

| 组件 | 版本 | 说明 |
|------|------|------|
| Node.js | >= 20 | 推荐 24 |
| pnpm | >= 10 | 包管理 |
| EADAF | 开发环境 | 运行于 `http://localhost:9527` |
| 测量工具/串口硬件 | 可选 | 无硬件时用 mock driver |

---

## 2. EADAF 侧准备（一次性）

1. **应用注册**：确认 FMMS 应用已注册（code=FMMS, id=`10000000-0001-4000-8000-000000000006`）
2. **数据建模**：按 `docs/EADAF数据结构设计-FMMS.md` §4 §5 在 EADAF 中补充/创建实体（用 EADAF AI 建模）
3. **采集管道**：创建 `fmms/measurementIngest` 管道，将 FMMS 应用 id 加入 allowlist
4. **API 访问策略**：将 `production/*` 服务设为「无限制」
5. **SSO 配置**：在 FMMS 应用 SSO 设置中配置签名盐值、回调 URI（`http://localhost:5180/auth/callback`）、跳转模式
6. **app_secret**：在密钥管理生成 app_secret，填入 `.env`（`FMMS_APP_SECRET`）
7. **Token 引导**：确认 `/api/v1/applications/token` 可用方式

## 3. 本地开发启动

### 3.1 环境变量
```bash
cp .env.example .env
# 填写 FMMS_APP_SECRET、SSO_JWT_SALT
```

各子包会读取根 `.env` 或各自 `.env`（见各包 `.env.example`）。

### 3.2 启动顺序
```bash
# 1. 安装依赖
pnpm install

# 2. (确保 EADAF 已运行)

# 3. 启动全部（并行）
pnpm dev
```

或分别启动（推荐联调时分窗观察日志）：
```bash
# 终端1：BFF (admin/backend)
cd admin/backend && pnpm dev          # http://localhost:5180

# 终端2：前端 (admin/frontend)
cd admin/frontend && pnpm dev         # http://localhost:5181

# 终端3：工位客户端 (Electron)
cd app-workstation-client && pnpm dev # 弹出 Electron 窗口
```

### 3.3 端口分配

| 服务 | 端口 |
|------|------|
| EADAF | 9527 |
| admin/backend (BFF) | 5180 |
| admin/frontend | 5181 |
| Electron | — |

## 4. Electron 联调

### 4.1 首次启动配置
1. Electron 启动 → OP_BASE 未配置 → 自动打开配置页
2. 填写 admin/frontend 地址（`http://localhost:5181`）、EADAF 地址、SSO 应用 ID
3. 点「测试连接」→ 保存
4. 主窗/副窗加载 OPWeb

### 4.2 单显示器调试
```bash
cd app-workstation-client && pnpm dev -- --small-secondary
# 副窗以小窗浮动，便于单屏开发
```

### 4.3 DevTools
- 主窗/副窗：`Ctrl+Shift+I`（webview DevTools）
- 配置页：同上

## 5. Mock PCS 种子数据

第一阶段不接真实 PCS。Mock 方式：
- 种子工卡/零部件/WBS 数据写入 EADAF 实体（`source=mock`）
- BFF `mockPCS` 服务定时查询 `source=mock` 且未分发的 WorkCard，按 `workstation_type` 经 WS `workcard.pushed` 推送到各工位

### 种子数据脚本（BFF 提供）
```bash
cd admin/backend && pnpm seed
# 向 EADAF 写入：
#   - 5 个 Workstation（OP01-OP04，含 OP02-1/OP02-2）
#   - 若干 WorkCard（各工位类型各 2-3 张）
#   - 对应 WorkCardPart / InspectionItem / WorkStep
#   - 1-2 个 TrayTemplate + TraySlot（分拣用）
#   - 1-2 个 LineStock + LineStockItem（收发用）
#   - 几个 MeasuringTool（含连接参数，用于采集联调）
```

## 6. 采集联调（无硬件时）

Electron 采集服务内置 **mock driver**：
```bash
# 配置页或环境变量启用 mock
USE_MOCK_DRIVER=true pnpm dev
# mock driver 定时发送随机数值帧 → BFF → EADAF ingest → WS 回显 → 副屏检测表填充
```

有真实硬件时：
1. 在 EADAF MeasuringTool 配置 protocol/port/baud_rate 等
2. Electron 采集服务自动发现并连接
3. 副屏故检工作台观察自动采集

---

## 7. SSO 联调

1. 确认 EADAF 中 FMMS 应用 SSO 已配置（盐值、回调 URI）
2. `.env` 的 `SSO_JWT_SALT` 与 EADAF SSO 盐值一致
3. OPWeb 点登录 → 跳 EADAF → 登录 → 回调
4. 验证：`GET /auth/me` 返回用户信息；路由受 AuthGate 保护

**常见问题：**
- 回调地址不匹配 → 检查 EADAF SSO 重定向 URI 与 BFF `SSO_CALLBACK_URL`
- JWT 校验失败 → 检查盐值是否一致
- CORS → 检查 BFF `BFF_CORS_ORIGIN`

## 8. 验收检查清单（P1 最小闭环）

打通「登录→选故检工位→看到 Mock 工卡→副屏自动采集一条数据写 EADAF」：

- [ ] EADAF 实体已建（§4 §5）
- [ ] 采集管道已建且 FMMS 在 allowlist
- [ ] BFF 启动，应用 token 获取成功
- [ ] 前端 OPWeb 启动
- [ ] Electron 配置 OP_BASE 后双屏加载 OPWeb
- [ ] SSO 登录成功，AuthGate 放行
- [ ] 工位选择故检1，持久化
- [ ] 工卡列表显示 Mock 工卡（故检类型）
- [ ] 接受工卡，状态更新
- [ ] 副屏检测表显示检测项（含标准值/公差）
- [ ] 采集（mock 或真实）一条数据 → 自动填充 + 判异
- [ ] EADAF InspectionRecord 有新记录（验证写入）
